# reproduce.sh Generation and Maintenance

## Objective

Generate and maintain `reproduce.sh` — the single executable entry point that reproduces ALL paper results. In PaperBench Full mode, this file is the most important artifact: the official grading pipeline executes it in a fresh container to generate results.

---

## Required Inputs

- `analysis_workspace/experiment_manifest.json` — list of all experiments with scripts, configs, dependencies
- `analysis_workspace/paper_analysis.json` — paper metadata
- `code_workspace/{name}/requirements.txt` — Python dependencies
- `code_workspace/{name}/scripts/` — all experiment scripts

---

## Required Output

### `code_workspace/{name}/reproduce.sh`

An executable shell script at the repo root. Structure:

```bash
#!/usr/bin/env bash
set -euo pipefail

# ===================================================================
# reproduce.sh — Reproduces all results for: {paper_title}
# Generated by pAI-Replicator
# ===================================================================

# --- Environment setup ---
echo "=== Setting up environment ==="
pip install -r requirements.txt

# --- Configuration ---
MAX_STEPS="${MAX_STEPS:-0}"  # 0 = full run, >0 = smoke test
SEED="${SEED:-42}"
DEVICE="${DEVICE:-cuda}"
RESULTS_DIR="results"
mkdir -p "$RESULTS_DIR"

# --- Helper: run with optional step limit ---
run_experiment() {
    local script="$1"
    local config="$2"
    local output="$3"
    shift 3
    local extra_args=("$@")

    local step_args=""
    if [ "$MAX_STEPS" -gt 0 ]; then
        step_args="--max-steps $MAX_STEPS"
    fi

    echo "=== Running: $script ==="
    python "$script" \
        --config "$config" \
        --seed "$SEED" \
        --device "$DEVICE" \
        --output-dir "$RESULTS_DIR/$output" \
        $step_args \
        "${extra_args[@]}"
    echo "=== Done: $script ==="
}

# --- Experiments (in dependency order) ---

# Table 1: {description}
run_experiment scripts/run_table1.py configs/experiment_table1.yaml table1

# Table 2: {description}
run_experiment scripts/run_table2.py configs/experiment_table2.yaml table2

# Figure 3: {description}
run_experiment scripts/run_figure3.py configs/experiment_figure3.yaml figure3

# ... (one entry per experiment from manifest)

echo "=== All experiments complete ==="
echo "Results saved to: $RESULTS_DIR/"
```

## Key Requirements

### 1. Executable and portable
- Shebang: `#!/usr/bin/env bash`
- `set -euo pipefail` — fail fast on errors
- `chmod +x reproduce.sh` after writing
- No absolute paths — everything relative to repo root
- No hardcoded usernames, home dirs, or machine-specific paths

### 2. MAX_STEPS support (mandatory)
- `MAX_STEPS=0` (default): full training run
- `MAX_STEPS=N` (N>0): smoke test with N steps per experiment
- CPU verification uses `MAX_STEPS=1` to validate the script runs

### 3. Dependency order
- If experiment B depends on outputs of experiment A, A runs first
- Read `experiment_manifest.json → dependencies` to determine order
- Comment the dependency in the script: `# Requires: table1 results`

### 4. Idempotent
- Running `reproduce.sh` twice should produce the same results (given same seed)
- Each experiment saves to its own output directory
- No global state leaks between experiments

### 5. Self-contained
- `pip install -r requirements.txt` at the top handles all dependencies
- No `conda`, `apt-get`, or other system package managers (unless truly necessary — and documented)
- Data download scripts go BEFORE experiments that need them

## When reproduce.sh Is Generated

### Initial generation (Phase 6 — training infrastructure)
Create a minimal `reproduce.sh` with:
- Environment setup section
- `run_experiment` helper
- Placeholder: `# Experiments will be added in Phase 9`

### Update (Phase 9 — experiment scripts)
After creating all experiment scripts, regenerate `reproduce.sh` with:
- One `run_experiment` call per entry in `experiment_manifest.json`
- Correct dependency ordering
- Correct config file paths

### Final validation (Phase 13 — integration test)
Verify `reproduce.sh`:
- Runs with `MAX_STEPS=1` and exits 0
- Lists all scripts from manifest
- Has no absolute paths
- Is executable

---

## Common Pitfalls

1. **Missing data download**: If datasets need downloading, add a download step before experiments. Don't assume data is pre-downloaded.
2. **CUDA assumptions**: Use `DEVICE="${DEVICE:-cuda}"` so CPU testing works with `DEVICE=cpu`.
3. **Path assumptions**: Use `$(dirname "$0")` or just relative paths from repo root.
4. **Missing results directory**: Always `mkdir -p` before writing.
5. **Non-determinism**: Always pass `--seed` for reproducibility.
