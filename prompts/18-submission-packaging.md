# Phase 13 Addendum: Submission Packaging (paperbench_full only)

## Objective

Package the generated code into a valid PaperBench submission: a self-contained git repository with `reproduce.sh` at its root, no blacklisted references, all files tracked, and size within budget.

This runs as part of Phase 13 (integration test) in `paperbench_full` mode only.

---

## Step 1: Run Submission Validator

```bash
python {SKILL_DIR}/scripts/validate_submission.py {REPO_DIR}
```

The validator checks:
- `reproduce.sh` exists and is executable
- `requirements.txt` exists
- All source files are git-tracked (no untracked dependencies)
- No URLs from `blacklist.txt` appear in any source file
- Total repo size < 500MB
- No absolute paths in source files
- No hardcoded credentials or API keys

If the validator reports violations, fix them:
- **Blacklisted URL found**: Remove the URL/reference from the offending file
- **Untracked files**: Either `git add` them or remove the dependency
- **Missing reproduce.sh**: Run `prompts/17-reproduce-sh.md` to generate it
- **Size over budget**: Remove large binaries, use `.gitignore`

Re-run the validator until the report is clean.

---

## Step 2: Final reproduce.sh Verification

Run reproduce.sh in smoke-test mode:
```bash
cd {REPO_DIR} && MAX_STEPS=1 DEVICE=cpu bash reproduce.sh
```

Must exit 0. If it fails, fix the issue and re-run.

---

## Step 3: Export Submission

```bash
python {SKILL_DIR}/scripts/export_direct_submission.py {REPO_DIR} {WORKSPACE}/submission/
```

This creates a clean copy of the repository at `{WORKSPACE}/submission/`:
- Initializes a fresh git repo
- Copies all tracked files
- Ensures `reproduce.sh` is at root and executable
- Runs `git clean -fd` safety check (nothing should be removed since we only copied tracked files)

---
